# 18_코드 스멜 탐지 [Ask 모드] 보고서 - Prompting

## 1. 세션 목적

이번 세션의 목적은 코드 수정 없이 `UnitConverter.java`의 코드 스멜을 분석하고, 문제 있는 부분만 지정된 표 형식으로 정리하는 것이었다.

주요 요구는 다음과 같았다.

- `UnitConverter.java`의 코드 스멜 분석
- 코드 수정 금지
- 문제 있는 부분만 표 형식으로 정리
- 이번 세션 내용을 보고서 파일로 내보내기
- 사용 프롬프트를 포함한 Prompting 파일 작성

## 2. 사용 프롬프트 기록

```text

- 단일 main() 함수에 파싱·검증·변환·출력이 혼재 (SRP 위반)
- if-else 체인 (새 단위 추가 시 기존 코드 수정 필요, OCP 위반)
```

## 3. 프로젝트 탐색 기록

먼저 `UnitConverter.java` 파일 위치를 확인했다.

확인 결과 `UnitConverter.java`는 두 곳에 있었다.

```text
UnitConverter.java
src/main/java/UnitConverter.java
```

두 파일을 모두 읽어 요청된 점검 항목에 맞게 분석했다.

## 4. 분석 기록

루트의 `UnitConverter.java`는 콘솔 입력, 문자열 파싱, 숫자 변환, 단위 검증, 단위 변환, 콘솔 출력이 모두 `main()` 안에 들어 있는 구조였다.

또한 `meter`, `feet`, `yard` 단위 처리가 `if-else` 체인으로 구현되어 있어 새 단위를 추가할 때 기존 조건문을 수정해야 했다.

변환 계수인 `3.28084`, `1.09361`은 상수 이름 없이 직접 하드코딩되어 있었다.

입력 형식이 잘못되거나 숫자 변환이 실패하는 경우에 대한 예외 처리는 없었다. 지원하지 않는 단위가 들어와도 오류가 발생하지 않고 `meterValue = 0` 상태로 출력되는 문제도 확인했다.

반면 `src/main/java/UnitConverter.java`는 `FEET_PER_METER` 상수와 `IllegalArgumentException`을 사용하고 있었기 때문에, 요청된 기준에서 별도 문제 항목으로 보고하지 않았다.
