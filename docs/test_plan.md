# Test Plan: Meter to Feet Conversion

## 1. 목적

본 테스트 계획서는 선택 샘플 예제인 `meter -> feet` 변환을 기준으로, Java 21 환경에서 JUnit 5 기반 단위 테스트의 범위, 우선순위, 경계값, 예외 케이스, 커버리지 측정 전략을 정의한다.

기준 요구사항은 `docs/README.md`와 `docs/PRD.md`의 기본 단위 변환 계약이다.

- 핵심 비율: `1 meter = 3.28084 feet`
- 입력 계약: `unit:value`
- 기본 지원 단위: `meter`, `feet`, `yard`
- 음수 정책: `0` 미만 값은 거부
- 미지원 단위 정책: 동적 등록 전 미지원 단위는 거부

## 2. 테스트 대상 샘플

| 항목 | 내용 |
|---|---|
| 기능 이름 | `meter -> feet` 변환 |
| 대표 입력 | `meter:2.5` |
| 기대 정밀 변환값 | `8.202100 feet` |
| 변환 공식 | `2.5 * 3.28084 = 8.202100` |
| README 요구사항 | 기본 요구사항 4번, README 비율 기반 변환 |
| PRD 추적성 | `FR4`, `PRD 5.1`, `PRD 7.1 S3`, `RR1` |

## 3. 테스트 범위

### 3.1 Domain 단위 테스트

Domain 테스트는 변환 비율과 계산 정확도를 직접 검증한다.

| 우선순위 | 테스트 범위 | 검증 내용 |
|---|---|---|
| P0 | `meter -> feet` 비율 변환 | `1 meter = 3.28084 feet` 계약을 절대 오차 `0.00001` 이하로 검증 |
| P0 | 대표값 변환 | `meter:2.5`가 `8.202100 feet`로 계산되는지 검증 |
| P0 | 영값 변환 | `meter:0`이 `0 feet`로 계산되는지 검증 |
| P1 | 매우 큰 수 변환 | 큰 입력값에서도 `Infinity`, `NaN`, overflow성 결과가 발생하지 않는지 검증 |
| P1 | 입력 단위 제외 규칙 | `meter` 입력 결과에서 `meter` 대상 결과가 제외되는지 검증 |

### 3.2 Boundary 단위 테스트

Boundary 테스트는 입력 문자열 파싱, 오류 분류, 출력 포맷 계약을 검증한다.

| 우선순위 | 테스트 범위 | 검증 내용 |
|---|---|---|
| P0 | 정상 입력 파싱 | `meter:2.5`에서 단위 `meter`, 값 문자열 `2.5`가 보존되는지 검증 |
| P0 | 형식 오류 | `meter=2.5`처럼 `:`가 없는 입력이 `INVALID_FORMAT` 계열 오류로 분류되는지 검증 |
| P0 | 숫자 파싱 실패 | `meter:abc`가 `INVALID_NUMBER` 계열 오류로 분류되는지 검증 |
| P0 | 음수 입력 | `meter:-1`이 `NEGATIVE_VALUE` 계열 오류로 분류되는지 검증 |
| P0 | 미지원 단위 | `parsec:1.0`이 `UNSUPPORTED_UNIT` 계열 오류로 분류되는지 검증 |
| P1 | 콘솔 출력 계약 | 성공 출력이 `<originalValue> <originalUnit> = <convertedValue> <targetUnit>` 구조를 따르는지 검증 |

## 4. 경계값 케이스

| ID | 입력 | 분류 | 기대 결과 | 우선순위 |
|---|---|---|---|---|
| BV-01 | `meter:0` | 영값 변환 | 유효 입력으로 처리되고 feet 변환값은 `0` | P0 |
| BV-02 | `meter:999999999999999999999999` | 매우 큰 수 | 계산 결과가 정의된 숫자이며 `Infinity`, `NaN`, 런타임 overflow성 실패가 없어야 함 | P1 |
| BV-03 | `meter:-1` | 음수 입력 정책 | 변환 결과 없이 `NEGATIVE_VALUE` 계열 오류 | P0 |
| BV-04 | `meter:abc` | 소수점/숫자 파싱 실패 | 변환 결과 없이 `INVALID_NUMBER` 계열 오류 | P0 |
| BV-05 | `meter=2.5` | `:` 없는 형식 오류 | 변환 결과 없이 `INVALID_FORMAT` 계열 오류 | P0 |
| BV-06 | `parsec:1.0` | 없는 단위 | 변환 결과 없이 `UNSUPPORTED_UNIT` 계열 오류 | P0 |

## 5. 예외 및 특이 케이스

| ID | 케이스 | 기대 정책 | 우선순위 |
|---|---|---|---|
| EX-01 | 빈 입력 문자열 | 변환 결과 없이 형식 오류로 분류 | P1 |
| EX-02 | `meter:` | 값 누락 오류 또는 형식 오류로 분류하고 변환 결과 미생성 | P0 |
| EX-03 | `:2.5` | 단위 누락 오류 또는 형식 오류로 분류하고 변환 결과 미생성 | P0 |
| EX-04 | `meter:2.5.1` | 숫자 파싱 실패로 분류 | P0 |
| EX-05 | ` meter : 2.5 ` | trim 정책에 따라 `meter:2.5`와 동등하게 처리되는지 확인 | P2 |
| EX-06 | `METER:2.5` | 단위 식별자가 대소문자를 허용하지 않는다면 미지원 단위로 분류 | P2 |
| EX-07 | `meter:NaN` | 숫자처럼 파싱되더라도 유효 변환값이 아니므로 숫자 오류로 분류 | P1 |
| EX-08 | `meter:Infinity` | 무한대 입력을 허용하지 않고 숫자 오류 또는 값 범위 오류로 분류 | P1 |

## 6. JUnit 5 테스트 우선순위

| 우선순위 | 대상 | 목적 | 완료 기준 |
|---|---|---|---|
| P0 | README 비율 기반 변환 | 핵심 비즈니스 로직 보호 | `meter:1`, `meter:2.5`, `meter:0` 변환 테스트 통과 |
| P0 | 입력 오류 분류 | 잘못된 입력이 변환 결과를 만들지 않도록 보호 | 형식 오류, 숫자 오류, 음수, 미지원 단위 테스트 통과 |
| P0 | 원 입력 보존 | 출력 계약 회귀 방지 | 출력 문자열에 원 입력 값과 단위가 유지됨 |
| P1 | 큰 수 처리 | overflow 및 특수 숫자 위험 완화 | 큰 수 입력에서 정의된 실패 또는 성공 정책을 검증 |
| P1 | 출력 대상 선정 | 입력 단위 제외 규칙 보호 | `meter` 입력 시 `feet`, `yard`만 결과 대상에 포함 |
| P2 | 공백/대소문자 정책 | 사용자 입력 변형의 정책 고정 | trim 및 case-sensitive 정책이 문서화된 기대와 일치 |

## 7. 커버리지 목표

| 영역 | 목표 | 측정 기준 |
|---|---:|---|
| Domain | 95% 이상 | 단위 비율, `meter -> feet` 계산, 영값, 큰 수, 입력 단위 제외 규칙 |
| Boundary | 85% 이상 | `unit:value` 파싱, 오류 분류, 원 입력 보존, 콘솔 출력 포맷 |
| Regression | 항목별 1개 이상 | README 비율, 음수 정책, 숫자 파싱 실패, 미지원 단위, 원 입력 보존 |

Domain은 비즈니스 로직의 회귀 위험이 가장 크므로 95% 이상을 목표로 한다. Boundary는 입력 오류 분기와 출력 문자열 계약을 중심으로 85% 이상을 목표로 한다.

## 8. JaCoCo 측정 전략

Maven 기준 커버리지 측정 명령은 다음과 같다.

```shell
mvn clean test jacoco:report
```

또는 테스트 실행이 이미 완료된 경우 다음 명령으로 리포트를 생성한다.

```shell
mvn jacoco:report
```

생성된 HTML 리포트는 다음 경로에서 확인한다.

```text
target/site/jacoco/index.html
```

측정 시 확인 기준은 다음과 같다.

- `Domain` 패키지 또는 모듈의 line coverage가 95% 이상인지 확인한다.
- `Boundary` 패키지 또는 모듈의 line/branch coverage가 85% 이상인지 확인한다.
- `meter -> feet` 변환 테스트가 README 고정 비율 `3.28084`를 직접 검증하는지 확인한다.
- 오류 입력 테스트가 변환 결과를 생성하지 않는지 확인한다.
- 커버리지 미달 시 누락된 분기가 정상 경로인지 오류 경로인지 구분해 P0 테스트부터 보강한다.

## 9. 완료 기준

테스트 계획의 완료 기준은 다음과 같다.

- `meter:2.5` 입력이 `8.202100 feet` 의미의 정밀 변환값을 검증한다.
- `meter:0`, `meter:-1`, `meter:abc`, `meter=2.5`, `parsec:1.0` 케이스가 모두 테스트 계획에 포함된다.
- Domain 95% 이상, Boundary 85% 이상 커버리지 측정 방법이 명시된다.
- `mvn jacoco:report` 결과를 `target/site/jacoco/index.html`에서 확인할 수 있다.
