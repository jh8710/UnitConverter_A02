# 05. PRD 작성 보고서

## 1. 목적

이번 세션의 목적은 Phase 4에서 완성한 Epic, User Stories, Gherkin, 체크리스트를 기반으로 `UnitConverter_A02`의 제품 요구사항 문서(PRD)를 작성하고 보정하는 것이다.

산출물은 `docs/PRD.md`에 반영했다. 구현 코드, 클래스 설계, 메서드 시그니처, 패키지 구조, 빌드 스크립트는 작성하지 않았다.

## 2. 기준 자료

| 기준 자료 | 사용 목적 |
|---|---|
| `docs/requirements.md` | Phase 4 Epic, Journey, User Stories, Gherkin, 체크리스트 기준 확인 |
| `docs/PRD.md` | 이번 세션의 최종 PRD 반영 대상 |
| `docs/README.md` | Phase 6 README 이관 상태와 누락 위험 비교 |
| `report/04_Epic → Journey → Story → Gherkin → 체크리스트_보고서.md` | Phase 4 의사결정 흐름 확인 |

## 3. 진행 단계

### 3.1 전체 PRD 초안 확인 및 보정

처음 요청은 Phase 4 산출물을 기반으로 PRD의 1~8번 섹션을 구조화하는 것이었다.

PRD는 다음 목차로 정렬했다.

| 섹션 | 내용 |
|---:|---|
| 1 | 프로젝트 개요 |
| 2 | 사용자 및 이해관계자 |
| 3 | 기능 요구사항 |
| 4 | 비기능 요구사항 |
| 5 | 데이터 요구사항 |
| 6 | 출력 요구사항 |
| 7 | 성공 지표 |
| 8 | 용어 정의 |

기존 PRD에서 테스트 가능성이 약한 표현을 확인하고, 변환 정확도 기준을 절대 오차 `0.00001` 이하로 고정했다.

### 3.2 섹션 1~2 작성

두 번째 요청은 Phase 4 Epic을 기반으로 PRD의 1~2번 섹션만 작성하는 것이었다.

반영 내용은 다음과 같다.

| 항목 | 반영 내용 |
|---|---|
| 한 줄 목적문 | What / Who / Why 형식으로 재작성 |
| 배경 및 문제 정의 | 관찰 관점의 문제 정의로 정리 |
| 목표 | 측정 기준이 있는 5개 목표로 구성 |
| 비목표 | 구현 산출물, 저장소 범위 밖 기능, 물리 단위 카탈로그 확장 제외 |
| 사용자 | Java/클린 아키텍처 학습자 페르소나 유지 |
| 사용 시나리오 | Phase 4 Journey의 문제 인식, 계약 정의, 도메인 분리, 검증, 회귀 보호 흐름 반영 |

### 3.3 섹션 3 기능 요구사항 작성

세 번째 요청은 Phase 4 User Stories를 기반으로 PRD 섹션 3을 작성하는 것이었다.

기능 요구사항은 Story 1~6과 연결되도록 재정렬했다.

| Story | PRD 기능 반영 |
|---|---|
| S1 입력 검증 | `unit:value`, 빈 문자열, 숫자 파싱, 음수 거부, 결과 미생성 |
| S2 레지스트리·OCP | 기본 단위, 등록 단위 기준 판단, 출력 대상 선정 |
| S3 환산 정확도 | README 비율, meter 허브 교차 변환, 입력 단위 제외 |
| S4 출력 포맷 | 콘솔, JSON, CSV, Table 출력 계약 |
| S5 설정 로드 실패 | 형식 오류, 필수 값 누락, 숫자 오류, 0 이하 비율, 입력 검증 실패와 구분 |
| S6 동적 단위 등록 | `register:<unit>:<metersPerUnit>`, `cubit`, 중복 단위명 거부 |

우선순위는 필수, 권장, 선택으로 구분했다.

| 우선순위 | 포함 기능 |
|---|---|
| 필수 | 입력 검증, 숫자 값 검증, 기본 단위, README 비율, meter 허브 변환, 미지원 단위, 콘솔 기본 출력 |
| 권장 | JSON 출력, CSV 출력, 설정 로드 실패 분류 |
| 선택 | Table 출력, 동적 단위 등록, 중복 단위명 등록 정책 |

입출력 계약은 문자열 수준으로만 정의했다. 코드, 클래스, 메서드, 빌드 설정은 작성하지 않았다.

### 3.4 PRD와 Phase 4 비교 보고

네 번째 요청은 완성된 PRD와 Phase 4 Epic, Story, Gherkin, 체크리스트를 비교하는 것이었다.

보고 결과는 다음과 같다.

| 비교 항목 | 결과 |
|---|---|
| Phase 4 Story에 있으나 PRD에 빠진 기능 | 뚜렷한 누락 없음 |
| PRD와 Phase 4 Gherkin 불일치 | 오차 수치, 0 값 정책, 콘솔 반올림, 출력 포맷별 실패 계약, 동적 등록 중복 정책, 설정 실패 세부 조건 |
| PRD → Phase 6 README 이관 위험 | 절대 오차, 0 값 정책, Table 출력 스키마, 중복 단위명 거부, 설정 실패와 입력 검증 실패 구분, FR 번호 참조 |

비교 요청은 보고만 수행했으며 파일 수정은 하지 않았다.

## 6. 검증 결과

`docs/PRD.md` 수정 후 문서 진단을 확인했으며 linter 오류는 발견되지 않았다.

구현 금지 조건을 지키기 위해 Java 코드, 클래스 설계, 메서드 시그니처, 빌드 스크립트는 작성하지 않았다.
